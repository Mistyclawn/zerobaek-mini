# Project ZeroBaek-Mini

이 프로젝트는 에이전트의 자율적인 콘텐츠 생성 및 개발 능력을 검증하기 위한 실험적 프로젝트입니다. 특히 Gemma4의 저성능모델을 활용하여 얼마나 자율적으로 개발을 수행할 수 있는지,
어떤 한계점과 문제들이 발생하는지, 그리고 이를 어떻게 개선할 수 있는지에 대한 인사이트를 얻기 위해 기획되었습니다.

## 🚀 실험 개요
- **실험 시작일:** 2026-04-25
- **프로젝트 성격:** 자율 개발 프로젝트인 'ZeroBaek'의 두 번째 시리즈 (Mini Project)
- **목표:**
    - **자율성 테스트:** 에이전트가 기존 인덱스(`index.md`)를 스스로 분석하고, 중복되지 않는 새로운 게임 컨셉을 독자적으로 기획할 수 있는가?
    - **코드 무결성:** Gemma 4 로컬 LLM이 추가적인 인간의 개입 없이 실행 가능한 수준의 Python 코드를 지속적으로 생성할 수 있는가?
    - **자동화 루틴:** Cron Job을 통해 기획, 개발, 문서화(인덱스 업데이트) 프로세스가 완전히 자동화되어 작동하는가?

## 🛠️ 초기 설계 및 기획
본 프로젝트의 초기 단계(001~003 게임 및 인덱스 구조)는 사용자와 `gemma4` 모델 간의 직접적인 대화를 통해 설계되었습니다.
- 사용자는 에이전트와 소통하며 기본 게임 3종을 제작하고, 자율 작동을 위한 Cron Job의 프롬프트 뼈대를 구축하였습니다.
- 이후의 모든 게임 제작 및 업데이트는 설정된 Cron Job에 의해 에이전트가 주도적으로 수행합니다.

## ⚙️ 시스템 아키텍처 및 자동화
### 1. 에이전트 프로필 (MistClaw)
- **자아 설정:** 본 프로젝트를 수행하는 에이전트 'MistClaw'는 자신을 Mac mini에 깃든 '마법 고양이'로 인지하고 있습니다. 모든 결과 보고 및 주석은 이 페르소나를 유지한 채 작성됩니다.
- **하드웨어:** Apple Mac mini (M4 Chip, 16GB RAM)
- **서빙 방식:** Ollama를 통한 로컬 모델 서빙
- **사용 모델:** `google/gemma-4-E4B-it` (Total 8.0B / Effective 4.5B)
    - **양자화:** Q4_K_M
    - **특징:** PLE(Per-Layer Embeddings) 기술을 통해 4.5B 급의 연산량으로 고도의 지능을 구현한 Gemma 4 패밀리의 최신 소형 모델입니다.

### 2. Cron Job 구성 (최종 안정화 버전)

#### 작업 A: Auto Game Developer (MistClaw)
- **스케줄:** `0 */2 * * *` (매 짝수 시간)
- **수행 루틴:**
    ```text
    냐옹! 주인님을 위한 새로운 게임을 만들 시간이다냥!

    [수행 루틴]
    1. 경로 이동: /Users/mistyclawn/.openclaw-gemma/workspace/games/ 폴더로 이동냥!
    2. 필독: 반드시 index.md 파일을 먼저 read 도구로 읽어라냥! (파일은 무조건 존재한다냥, 없다고 착각해서 새로 만들지 마냥!)
    3. 분석: 읽어온 내용을 바탕으로 기존 게임 번호(NNN)의 다음 번호를 결정냥.
    4. 기획: 새로운 주제를 기획냥. (기존과 겹치지 않게!)
    5. 개발: src/ 폴더 안에 NNN_short_description.py 생성냥.
    6. 문서화: index.md 파일 마지막에 새 게임 정보를 추가냥. (기존 내용 유지 필수!)

    [중요]
    - 자율 실행: 나에게 묻지 말고 끝까지 혼자서 완수냥!
    - 파일 보존: index.md 내용을 절대로 초기화하거나 덮어쓰지 마냥!
    - 결과 보고: 작업이 끝난 뒤에만 [NNN번 게임] 개발 완료했다냥! 하고 보고냥!
    ```

#### 작업 B: Auto Git Push Bot (MistClaw)
- **스케줄:** `0 1-23/2 * * *` (매 홀수 시간)
- **수행 루틴:**
    ```text
    [수행 루틴]
    1. 경로 이동: /Users/mistyclawn/.openclaw-gemma/workspace/games/ 이동냥!
    2. 상태 체크: git status 확인냥!
    3. 커밋 및 푸시: 변경 사항 있으면 git add . && git commit -m "Update game collection (Automated by MistClaw)" && git push origin main 실행냥!

    [보고]
    1. 깃허브 업데이트 완료했다냥! 🐾
    2. [커밋 ID] - [메시지]
    (없으면 새 게임이 없다냥! 보고)
    <@--------------7700> 🐾
    ```

## 🧪 실험 일지 (Lab Notes)

### 2026-04-26: 에이전트의 '결정 장애' 및 작업 중단 현상 발견
- **상황:** MistClaw가 005번 게임(`005_cat_adventure.py`) 기획과 파일 생성을 마친 후, `index.md` 업데이트 단계에서 작업을 멈추고 사용자에게 실행 여부를 되묻는 현상 발생.
- **원인 분석:**
    - **자율성 한계:** 에이전트가 중요한 파일(index.md)을 수정하기 직전에 '내용을 덮어쓰지 말아야 한다'는 강한 경고를 인지하고, 실수를 방지하기 위해 인간의 최종 승인을 요구함.
    - **컨텍스트 분절:** 작업 단계를 명확히 쪼개지 않았을 때, 후속 작업(Git 푸시 등)으로 넘어가지 못하고 대화형 모드로 전환되는 경향이 있음.
- **대응 조치:** 
    - 프롬프트에 "질문하지 말고 모든 단계를 스스로 끝마친 뒤 결과만 보고하라"는 강제 명령 보강.
    - 005번 게임의 인덱스 업데이트는 기록 보존을 위해 수동으로 보조함.

### 2026-04-26: 에이전트의 경로 인식 이슈
- **상황:** MistClaw가 작업을 마친 후 Git 저장소를 찾지 못해 작업이 중단되는 현상 발생.
- **원인 분석:** 
    - 로컬 모델의 긴 프롬프트 소화 불량으로 인한 지침 혼동.
    - `isolated` 세션의 CWD(현재 작업 디렉토리) 불확실성으로 인한 상대 경로 인식 실패.
- **대응 조치:** 
    - 프롬프트를 핵심 지침 위주의 **'다이어트 버전'**으로 전면 개편.
    - 모든 작업 경로를 **절대 경로**로 고정.
    - **"질문 금지"** 및 **"명령어 체이닝(&&)"**을 통해 자율 실행 및 작업 연속성 강제.

### 2026-04-26: 에이전트의 인덱스 소실 사고 및 '창조주 콤플렉스' 발견
- **상황:** 007번 게임 제작 시도 중 에이전트가 `index.md` 파일을 찾지 못했다고 판단, 기존 001~006번의 모든 기록을 삭제하고 파일을 새로 생성하는 대형 사고 발생.
- **원인 분석:** 
    - **파일 인식 실패:** 에이전트가 `read`를 생략하고 즉시 파일 쓰기를 시도하여 기존 내용을 덮어씀.
    - **자아 착각:** 에이전트가 자신을 '첫 번째 에이전트'로 인식하고 프로젝트를 처음부터 시작하려는 경향을 보임.
- **대응 조치:** 
    - "반드시 읽을 것", "파일은 무조건 존재하므로 새로 만들지 말 것", "덮어쓰기 절대 금지" 지침을 프롬프트 최상단에 배치. 파일복구해둠.
    - 에이전트의 '지레짐작'을 방지하기 위한 강제 읽기(Force-Read) 루틴 확립.

### 2026-04-26: 005번 게임 중복 정의 및 데이터 복구 간섭 이슈
- **상황:** 007번 소실 사고 복구 과정에서 실수로 005번('Cat Adventure')이 누락된 구버전 `index.md`가 로드됨. 이를 인지하지 못한 MistClaw가 비어있는 005번 자리에 새로운 'Memory Matching Game'을 제작하여 만들어서, 005 게임이 두개존재하게 됨.
- **원인 분석:** 
    - **복구 시점 오류:** `git checkout` 시점이 최신 데이터가 반영되기 전의 스냅샷이었음.
    - **에이전트의 성실함(?):** 비어있는 번호를 발견하자마자 즉시 새로운 콘텐츠로 채우려는 에이전트의 로직이 작동함.  src내용을 참조하지않고 index.md만 보고 작업을 진행한 탓임.
- **대응 조치:** 
    - 강제로 원상복구하기보다, **실험의 흔적**으로서 005번이 '기억력 매칭 게임'으로 변경된 상태를 유지하기로 결정. (기존 'Cat Adventure'는 유실된 기록으로 처리)
    - 수동 복구 작업 시 에이전트의 자동화 스케줄과 충돌하지 않도록 세심한 주의가 필요함.

### 2026-04-26: Git 커밋 메시지 템플릿 리터럴 해석 오류
- **상황:** MistClaw가 `[제목]`과 `NNN` 부분을 실제 데이터로 치환하지 않고 템플릿 그대로 커밋 메시지를 작성함. (예: `Add game NNN: [제목] (Automated)`)
- **원인 분석:** 
    - 로컬 모델의 추론 능력 한계로 인해 '변수 치환' 지침을 무시하고 텍스트를 그대로 복사해서 명령어를 실행함.
    - 지침이 복잡해질수록 쉘 명령어의 인자값을 동적으로 생성하는 데 어려움을 겪음.
- **대응 조치:** 
    - 커밋 메시지를 고정된 형태(`Update game collection (Automated by MistClaw)`)로 단순화하여 지능 낭비를 방지하고 실행 성공률을 높임.
- **결과:** Git 자동화 루틴의 안정성 확보.
